# Styling

- Use Tailwind v4 for product UI.
- Use utilities for layout, spacing, type, color, borders, radius, and normal states.
- Use `src/styles.css` for `@theme`, base, fonts, globals, keyframes, third-party overrides, prose, and awkward selectors.
- Prefer components or partials for repeated UI.
- Do not add CSS classes only to hide long utility strings.
- Turn repeated arbitrary values into a token, utility, or component API.
- Leave true one-offs inline.
- Keep CSS small for product UI.
- Use more CSS for content-heavy or generated markup.
